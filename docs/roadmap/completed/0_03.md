## SAHPool OPFS ✅

Plan:
[plans/completed/plan_sahpool_opfs.md](../plans/completed/plan_sahpool_opfs.md)

Replaced `StorageAdapterWeb` with `StorageAdapterSahPool`, routing all OPFS I/O
through a Web Worker using `FileSystemSyncAccessHandle` for 3–4× throughput.
Per-op handle lifecycle (open → write → flush → close) provides durability. Real
cross-tab exclusion via exclusive session-held lock handle.
