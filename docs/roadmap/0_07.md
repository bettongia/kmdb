# v0.07

## Encryption

Proposal: [Technical Proposal: Database Encryption](../proposals/encryption.md)

Look at encryption options for the local and shared data.

Consider Per-platform secure storage (iOS Keychain, Android SharedPreferences,
web `localStorage`, desktop app-data dir) injectable via a `PlatformIdStore`
interface; default falls back to `$meta`.

Consider encryption at the value level in the KV store. This will need to be
very user-friendly and could follow an approach of the user selecting their own
salt rather than approaches such as managing PGP keys. The approach needs to
work in the environment where the user understands little about encryption and
won't store things like private keys right next to the KMDB data in their chosen
storage location (e.g. local directory or Google Drive)
