# v0.05

> Review [KMDB Roadmap Review — 2026-06-05](../reviews/roadmap-review-2026-06-05.md)
> before planning any work in this release.

v0.05 makes KMDB's native-binary dependencies work across all target platforms
and establishes the CI signing pipeline. It is a **hard prerequisite** for any
feature work in `betto_inferencing` on mobile, and for web sync correctness
(WASM decompression, see §betto_zstd below).

Two packages require cross-platform binary support:

| Package | Mechanism | Note |
|---|---|---|
| `betto_zstd` | `native_toolchain_c` + WASM | Web decompression non-deferrable |
| `betto_inferencing` → `betto_onnxrt` | ONNX Runtime FFI | iOS/Android unblocked by extraction |

`betto_mediatype_detector` (used by `MediaTypeDetector`, replacing the former
`betto_registry`) is pure Dart — no binary work required. `betto_icu`
extraction is complete.

### [x] betto_zstd — WASM and multi-platform pipeline

Plan: [betto_zstd pipeline](../plans/plan_betto_zstd_pipeline.md) — work lives
in the [`bettongia/zstd`](https://github.com/bettongia/zstd) repo. **Complete**
(plan marked Implemented 2026-06-09).

- [x] WASM compression + decompression (Emscripten self-built WASM; frame
      compatibility guaranteed by construction — same C source as native path)
- [x] Frame-compat verification: golden-file test
      (`test/frame_compat_test.dart`) covers native→WASM, WASM→native, and WASM
      round-trip
- [x] Multi-platform CI (macOS, Linux x86_64, Android, iOS, Windows x64/MSVC,
      Web/Chrome)
- [x] `VERSION_ZSTD` file + build-hook assertion (parses `#define` from
      `zstd.h`)
- [x] pub.dev publishing (`publish_to: none` removed)

Q6 (vendored C source update automation) deferred to a follow-up.

Key constraints:

- [`zstandard`](https://pub.dev/packages/zstandard) is the candidate WASM
  back-end; frame compatibility must be verified empirically before adoption.
- Apple Archive does not support Zstd; `native_toolchain_c` is the approach for
  iOS/macOS.
- Android: [Zstd-jni](https://github.com/luben/zstd-jni) bundles pre-compiled
  `.so` for all major ABIs; added to the **host app's** `build.gradle`.

### [x] betto_onnxrt extraction and iOS/Android support

Plan: [Extract betto_onnxrt](../plans/completed/plan_betto_onnxrt_extraction.md)
— Proposal: [betto_onnxrt design](../proposals/implemented/betto_onnxrt.md) — **Complete**

Stages A and B complete (PR #42, 2026-06-10). iOS SPM shim
(`betto_onnxrt_ios`) and Android native-assets wiring are now published and
in place; `betto_onnxrt: ^0.1.0-dev.1` is pinned in the workspace-root
`dependency_overrides` (commit a1b4d9e, 2026-06-17).

- [x] Stage A: generic `betto_onnxrt` package scaffold + `ModelDownloader`
      (`github.com/bettongia/onnxrt`, CI green on macOS/Linux/Windows)
- [x] Stage B: replace in-tree `betto_inferencing` types; `UnsupportedError`
      documented for iOS pending SPM shim
- [x] iOS: ORT XCFramework via SPM — handled by `betto_onnxrt_ios: ^0.1.0-dev.1`
      (published to pub.dev). A Flutter plugin (requires Flutter ≥ 3.27.0);
      Flutter's SPM tooling resolves and statically links ORT 1.24.2;
      `OnnxRuntime.load()` uses `DynamicLibrary.process()` — no CocoaPods or
      separate dylib loading. Consumer Flutter app adds this package alongside
      `betto_onnxrt`; kmdb workspace is unaffected (Dart-only). Plan:
      [`plan_ios_spm_shim.md`](https://github.com/bettongia/onnxrt/blob/main/plans/plan_ios_spm_shim.md)
- [x] Android: native-assets build hook in `betto_onnxrt` downloads the correct
      ORT `.so` from Maven Central for each target ABI (`arm64-v8a`,
      `armeabi-v7a`, `x86_64`, `x86`), validates via SHA-256, and embeds it in
      `lib/{abi}/` — no manual AAR wiring needed. Only host-app requirement:
      `minSdkVersion 35` in `android/app/build.gradle`.
- [x] `VERSION_ONNX` file + `tool/generate_versions.dart` → `versions.g.dart`
- [x] Multi-platform CI (macOS, Linux x86_64, Windows x64)
- [x] pub.dev publishing (`betto_onnxrt: ^0.1.0-dev.1`)

Key constraints:

- **iOS:** Use Swift Package Manager (`onnxruntime-c` XCFramework). Do **not**
  use CocoaPods (deprecated). Do **not** use `onnxruntime-mobile` — reduced
  opset breaks the existing BGE model.
- **Semantic search is opt-in on iOS** — ORT framework adds ~10–15 MB
  post-thinning; model downloads on first use.
- **ONNX models are data assets, not executables** — downloaded and cached on
  first use on all platforms, including iOS (App Store permits data downloads).
  Vector indexes are local-only, excluded from sync.
