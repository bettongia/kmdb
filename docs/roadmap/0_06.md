# v0.06

**NOTE** Ensure you review
[KMDB Roadmap Review — 2026-06-05](../reviews/roadmap-review-2026-06-05.md) as
part of any planning work.

Support traditional full-text search and vector-based searches in the vault.
This work will look to also embrace a more inclusive range of languages (not
just English) and should improve search for database document indexes as well.

See: [Technical Proposal: Vault File Search](../proposals/vault_search.md)

References:

- [SQLite FTS5](https://www.sqlite.org/fts5.html)
- [Postgres Full Text Search](https://www.postgresql.org/docs/current/textsearch.html)

## Review the mimetype detection approach

We also struggle to use mime detectors to detect packages file types such as
MSOffice etc - things that are wrapped in a container such as a Zip archive.
Tika provides for coded detectors as well as the magic and file name detectors.

Google's [Magika](https://github.com/google/magika) could help avoid writing a
bunch of detectors - see
[Magika: AI powered fast and efficient file type identification](https://opensource.googleblog.com/2024/02/magika-ai-powered-fast-and-efficient-file-type-identification.html).
The approach uses ONNX which aligns with the runtime used in the
betto_inferencing package.

## Detect the charset in a file

Vault doesn't currently undertake any checking of the charset for pertinent
files. This is fine if we're just uploading/downloading the file but efforts to
read the file (e.g. for extracting plain text) is likely to need to be charset
aware.

## Detect the language used in text

We don't have a great story around i18n at the moment - it's all very English
oriented.

Some options to consider:

- [Google AI Edge - Language detection](https://ai.google.dev/edge/mediapipe/solutions/text/language_detector) -
  relies on tflite
- [Floret](https://github.com/explosion/floret) is a maintained fork of
  [Meta's Fasttext](https://huggingface.co/facebook/fasttext-language-identification)
  - [Home page](https://fasttext.cc/docs/en/language-identification.html) -
    links to the smaller downloads.
- [Lingua](https://github.com/pemistahl/lingua-rs) is often used for shorter
  texts
  - Written in Rust

It looks like a mix of Lingua for shorter texts and another model (e.g. Floret)
for longer texts is the best approach.

## Consider the embedding model used in text search

`betto_inferencing` uses the
[BGE Small En v1.5](https://huggingface.co/BAAI/bge-small-en-v1.5) model and is
limited to English. Consider alternatives such as
[BGE-M3](https://huggingface.co/BAAI/bge-m3) which support multiple languages.

### Model identity and index invalidation

`$vec:` namespaces are local to each device and excluded from sync. This means
each device independently builds its vector index from whatever model it has
downloaded. When the model changes (e.g. BGE Small En → BGE-M3), the existing
`$vec:` index was built with different embedding geometry and queries against it
will return nonsense — the index must be invalidated and rebuilt.

This requires recording model identity in the `$meta` namespace so that on
startup (or on model download) the stored model version can be compared to the
active model, and a full index rebuild triggered if they differ.

This work is a **prerequisite** for any model migration: it must land before the
embedding model is changed in production, regardless of which model is chosen.
